# FL (Federated Learning)

연합학습(Federated Learning) 기반 객체 탐지 프로젝트입니다.  
데이터는 중앙 서버로 이동하지 않고 각 클라이언트에 유지되며, 모델만 공유하면서 학습이 진행됩니다.

# Data Flow

1. Server 초기 모델 생성
   - 중앙 서버가 YOLO 기반 글로벌 모델을 생성합니다.

2. Client로 모델 배포
   - 서버는 여러 클라이언트에게 현재 글로벌 모델을 전송합니다.

3. 로컬 데이터 학습
   - 각 클라이언트는 자신이 보유한 이미지 데이터(Labeled / Unlabeled)를 사용해 로컬 학습을 수행합니다.
   - 원본 데이터는 외부로 전송되지 않습니다.

4. 학습 결과 업로드
   - 클라이언트는 업데이트된 가중치(Parameter) 또는 Gradient만 서버에 전달합니다.

5. Server 통합
   - 서버는 여러 클라이언트의 결과를 평균(FedAvg) 등 방식으로 통합하여 글로벌 모델을 업데이트합니다.

6. 반복 학습
   - 업데이트된 모델을 다시 클라이언트에 배포하며 반복적으로 성능을 향상시킵니다.
